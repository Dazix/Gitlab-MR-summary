v2.1.0

2020/03/17

tl;dr

  • add fixture Show commits count in merge button
  • add fixture Expand CI/CD secrets textarea

  • allow add gitlab domain without token because of fixtures

Add Gitlab fixtures Show commits count in merge button, Expand CI/CD secrets textarea and Automatically check delete source branch in new-mergerequest page.
Because of these fixtures you can now add gitlab domain without token so only fixtures can be used without MR summary functionality. If you are used to squash commits before merging and sometimes you forget it, first fixture will help a little bit. The second fixture helps you with small unresizable textareas in Settings > CI/CD > Variables, now they will be bigger for better readability of values. And third one automatically check option delete branch when mergerequest is merged on mergerequest create.
And last thing, extension now detects when you merge a mergerequest and remove it from the summary.

v2.0.0

2020/03/11

tl;dr

  • local merge-requests data caching
  • background data downloading
  • tabs syncing
  • user actions (approve/revoke approve & merge-request creation)

Version 2.0.0 is really a major update. The whole logic has been rewritten so data doesn't have to be downloaded on every page load, but instead is downloaded in the background and stored locally for a user-specified time. When the user makes an update or change (approve / disapprove), the data is synchronized between all Gitlab tabs, so there is no need to reload the page on any tabs already open.
